## 1. 项目背景

### 1.1 业务背景

影视短片、短剧、广告宣传等视频内容的创作需求日益增长，传统视频制作流程存在以下痛点：

- **门槛高**：从剧本到成片需要编剧、导演、摄影、剪辑等多角色协作，普通创作者难以独立完成
- **周期长**：分镜设计、素材拍摄、后期剪辑等环节耗时巨大，一条短视频往往需要数天甚至数周
- **成本高**：演员、场地、设备、后期等成本高昂，试错代价大
- **角色一致性难**：AI生成视频时，同一角色在不同镜头中外观容易发生变化，难以保持一致

### 1.2 项目定位

构建一个**AI视频生成智能体系统**，实现"剧本输入 → 智能解析角色/场景 → 好莱坞级分镜生成 → 静态/动态提示词生成 → 视频片段生成 → 下载"的全链路自动化，让创作者仅需输入一段剧本即可获得可编辑、可下载的视频片段。

### 1.3 项目目标

| 目标 | 描述 | 指标 |
|------|------|------|
| 剧本解析 | 从剧本中提取角色与场景，生成一致性提示词 | 角色识别准确率≥90% |
| 分镜生成 | 结合好莱坞导演视听语言生成专业分镜表 | 单次生成≤15秒 |
| 静态提示词 | 去除运动信息生成静态画面提示词 | 用于高质量定帧图生成 |
| 动态提示词 | 基于静态提示词叠加运动生成动态提示词 | 运动描述完整率≥95% |
| 视频生成 | 调用ComfyUI/API生成视频片段 | 单片段生成≤5分钟 |
| 全流程可编辑 | 每个环节支持手动编辑修改 | 编辑后可重新触发下游 |
| 多模型支持 | LLM支持API/Ollama，图片视频支持API/ComfyUI | 双通道无缝切换 |

## 2. 用户角色与场景

### 2.1 用户角色

| 角色 | 描述 | 核心需求 |
|------|------|----------|
| 视频创作者 | 独立短视频/短剧创作者 | 快速从剧本生成视频片段，降低制作门槛 |
| 编导人员 | 影视/广告编导 | 获取专业分镜方案，可手动调整优化 |
| 系统管理员 | 系统运维与配置管理 | 管理模型配置（API/Ollama/ComfyUI）、监控系统 |

### 2.2 核心使用场景

**场景一：全自动流程**
> 创作者在对话框中输入剧本 → 系统自动解析角色和场景提示词 → 生成好莱坞级分镜表 → 生成静态提示词表 → 生成动态提示词表 → 逐片段生成视频 → 一键下载全部或单独下载。

**场景二：分步编辑流程**
> 创作者输入剧本后，在角色提示词阶段手动修改某角色的外貌描述（如"红色长发"改为"金色短发"）→ 确认后继续生成分镜 → 在分镜表手动调整某镜头的景别和运动 → 重新生成受影响的静态/动态提示词 → 生成视频。

**场景三：本地模型生成**
> 管理员配置本地Ollama作为LLM、本地ComfyUI作为图片/视频生成引擎 → 创作者输入剧本 → 全流程使用本地模型完成，无需外部API。

**静态提示词生成规则：**

```
静态提示词 = 场景提示词 + 角色一致性提示词 + 画面内容(去除运动描述) + 景别 + 角度 + 光影氛围 + 画质修饰词
```

```json
{
```

### 3.5 F4. 动态提示词模块

| 编号 | 功能 | 描述 | 优先级 |
|------|------|------|--------|
| F4.1 | 动态提示词生成 | 基于静态提示词，按分镜运动信息生成动态提示词 | P0 |
| F4.2 | 动态提示词表展示 | 前端以表格形式展示动态提示词，与分镜表对应 | P0 |
| F4.3 | 动态提示词手动编辑 | 支持编辑每个动态提示词 | P0 |
| F4.4 | 运动描述注入 | 将分镜的镜头运动转化为视频生成引擎可理解的运动指令 | P0 |

**动态提示词生成规则：**

```
动态提示词 = 静态提示词(画面部分) + 运动描述(镜头运动+角色动作) + 时长 + 视频生成引擎参数
```

**运动描述映射表：**

| 分镜运动 | 动态提示词描述 |
|----------|----------------|
| 推 | camera slowly zooms in / push in |
| 拉 | camera slowly zooms out / pull back |
| 摇 | camera pans left/right |
| 移 | camera tracks left/right |
| 跟 | camera follows subject |
| 升降 | camera tilts up/down, crane shot |
| 手持 | handheld camera, slight shake |
| 固定 | static camera, no movement |

```
         ▼
  ┌─────────────┐
  │ 静态提示词   │  去除运动，生成静态画面提示词
  │ 表生成       │  注入角色一致性提示词
  └──────┬──────┘
         │  （支持手动编辑）
         ▼
  ┌─────────────┐
  │ 静态图片生成 │  调用图片引擎生成定帧参考图
  └──────┬──────┘
         │
         ▼
  ┌─────────────┐
  │ 动态提示词   │  基于静态提示词+分镜运动信息
  │ 表生成       │  生成动态提示词
  └──────┬──────┘
         │  （支持手动编辑）
         ▼
  ┌─────────────┐
  │ 视频片段生成 │  动态提示词+静态参考图 -> 视频引擎
  └──────┬──────┘
         │
         ▼
  ┌─────────────┐
  │ 下载         │  单独下载 / 一键下载全部
  └─────────────┘
```

### 5.2 子流程：编辑后重生成

```
用户编辑某环节内容（如修改角色提示词/分镜/静态提示词）
        │
        ▼
  系统标记受影响的下游环节
        │
        ▼
  用户确认是否重新生成下游
        │
        ▼
  重新生成受影响的提示词/图片/视频
```

### 5.3 子流程：视频生成任务状态流转

```
PENDING -> RUNNING -> SUCCESS
                ↘ FAILED -> RETRY -> RUNNING
                ↘ CANCELLED
```

## 6. 约束与假设

### 6.1 约束

- ComfyUI本地部署需要GPU（建议≥8GB显存）
- Ollama本地部署需要足够内存和显存运行大模型
- 视频生成质量受LLM提示词质量和生成引擎能力双重影响
- 角色一致性依赖于静态参考图的生成质量

### 6.2 假设

- 用户已具备ComfyUI本地部署环境或相关API访问权限
- 用户已配置Ollama或拥有至少一个大模型的API Key
- 网络环境可稳定访问相关API服务（使用API模式时）

## 7. 术语表

| 术语 | 说明 |
|------|------|
| 分镜（Storyboard） | 将剧本拆分为多个镜头，描述每个镜头的画面和指令 |
| 景别 | 镜头取景范围，如远景、全景、中景、近景、特写 |
| 角度 | 摄影机拍摄角度，如平视、俯拍、仰拍 |
| 镜头运动 | 摄影机运动方式，如推、拉、摇、移、跟 |
| 静态提示词 | 去除运动信息的画面提示词，用于生成静态参考图 |
| 动态提示词 | 包含运动信息的提示词，用于生成视频片段 |
| 角色一致性 | 同一角色在不同镜头中保持外观一致 |
| ComfyUI | 基于节点的AI图像/视频生成工作流工具 |
| Ollama | 本地大模型运行框架 |
| LLM | 大语言模型（Large Language Model） |
| Agent | 智能体，能自主感知、决策、执行任务的AI系统 |
