# AI视频生成智能体系统 - 任务清单文档

> 版本：v1.0  
> 日期：2026-07-14  
> 状态：草案

---

## 1. 文档概述

本文档将系统需求分解为可执行的开发任务，按阶段和模块组织，包含任务编号、描述、优先级、预估工时、依赖关系和验收标准，作为项目开发的执行指南。

---

## 2. 里程碑规划

| 里程碑 | 名称 | 目标 | 预计周期 |
|--------|------|------|----------|
| M0 | 项目初始化 | 搭建项目骨架、基础设施、CI/CD | 第1周 |
| M1 | 剧本解析模块 | 实现剧本输入→角色/场景提示词生成 | 第2-3周 |
| M2 | 分镜生成模块 | 实现好莱坞导演分镜表生成与编辑 | 第4周 |
| M3 | 提示词模块 | 实现静态/动态提示词生成与编辑 | 第5周 |
| M4 | 视频生成模块 | 实现图片/视频生成、预览、下载 | 第6-7周 |
| M5 | 系统集成与优化 | 端到端集成、性能优化、文档完善 | 第8周 |

## 5. 关键任务验收标准

### 5.1 剧本解析验收标准（T1.9-T1.11）

- [ ] 输入一段≥500字的剧本，系统在15秒内返回解析结果
- [ ] 角色识别准确率≥90%（与人工标注对比）
- [ ] 角色提示词包含：外观、服装、性格、一致性标签、参考图提示词
- [ ] 场景提示词包含：地点、时间、天气、环境、光影、氛围
- [ ] 支持OpenAI API和Ollama两种LLM后端，结果一致

### 5.2 分镜生成验收标准（T2.3）

- [ ] 根据剧本+角色+场景生成分镜表，包含所有必填字段
- [ ] 分镜号连续编号，角度/景别/运动符合视听语言规范
- [ ] 单次生成≤15秒
- [ ] 分镜数量合理（1分钟剧本约8-15个分镜）
- [ ] 画面内容描述详细，台词和音效标注清晰

### 5.3 提示词生成验收标准（T3.3, T3.5）

- [ ] 静态提示词已去除所有运动描述
- [ ] 静态提示词自动注入角色一致性提示词
- [ ] 动态提示词包含完整的运动描述
- [ ] 动态提示词包含时长和运动配置
- [ ] 负面提示词配置完整

### 5.4 视频生成验收标准（T4.11-T4.12）

- [ ] 静态图片生成成功，分辨率符合设定
- [ ] 视频片段生成成功，时长与分镜设定一致
- [ ] 角色在不同镜头中外观基本一致
- [ ] 生成进度通过WebSocket实时推送
- [ ] 支持单独和一键下载

### 5.5 编辑功能验收标准（T5.2）

- [ ] 角色/场景提示词支持编辑保存
- [ ] 分镜表所有字段支持编辑
- [ ] 静态/动态提示词支持编辑
- [ ] 编辑后可触发下游重新生成
- [ ] 编辑操作有审计日志

| LLM输出JSON格式不稳定 | 解析失败 | 高 | 使用JSON修复库、增加重试机制、Prompt强化约束 |
| ComfyUI工作流节点变更 | 生成失败 | 中 | 工作流模板版本管理、节点兼容性检测 |
| 角色一致性效果不佳 | 视频质量差 | 高 | 多参考图策略、IPAdapter权重调优、参考图筛选 |
| 视频生成耗时长 | 用户体验差 | 中 | 异步任务+进度推送、批量生成、缓存复用 |
| Ollama本地模型性能不足 | 延迟高 | 中 | 支持快速切换到API模式、模型量化优化 |
| GPU显存不足 | 生成失败 | 中 | 分批生成、显存监控、降分辨率选项 |
